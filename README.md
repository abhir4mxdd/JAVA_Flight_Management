# JAVA_Flight_Management
A basic flight management system having a main flight record as base class flight manager as nested and flight classes as derived to view , update flight details

FlightManager is the main user in this system. Only he can update / view the flight details which is hidden from other common users.

VIP_Class , VVIP_Class nad PublicFlights are used only as an example to differentiate flight classes.
Their only use is their constructors to assign members some data.

FlightManager can enter data himself manually or I gave a random loop to add flights data for convinience of the dummy users.
